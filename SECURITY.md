# Security Policy

If you discover a security issue in VOIDcore, please avoid filing a public
issue with exploit details before maintainers have a chance to assess it.

For now, report security concerns privately to the repository owner through
GitHub security reporting if enabled, or by direct contact through the
repository profile.

When reporting, include:

- affected version or commit
- deployment method
- reproduction steps
- impact assessment
- any proof-of-concept details needed to validate the issue

Please do not include secrets, production credentials, or private user data
unless absolutely necessary.
